Οργάνωση αρχείων:
        - Τα αρχεία κώδικα είναι τοποθετημένα στον φάκελο source.
        - Τα αρχεία κεφαλίδας είναι τοποθετημένα στον φάκελο headers.
        - Τα αντικειμενικά αρχεία τοποθετούνται στον φάκελο bin κατα την μεταγλώττιση.
        - Αρχεία που μπορεί να δημιουργούνται απο το πρόγραμμα (πχ. output) τοποθετούνται στον κύριο κατάλογο.
        - Το makefile, το readme, το completion report και το εκτελέσιμο μετα την μεταγλώττιση βρίσκονται στον κύριο κατάλογο.

Οδηγίες κατά την εκτέλεση:

1) Δεν υποστηρίζεται η κλήση ενός alias με επιπλέον ορίσματα. Δηλαδή τα ενδεχόμενα ορίσματα θα πρέπει να αποθηκευτούν στην δημιουργία του alias.
2) Οι εντολές για δημιουργία και διαγραφή alias είναι : createAlias {aliasname} {command}, deleteAlias {aliasname}. Η εντολή πρέπει να περαστεί χωρίς εισαγωγικά.
3) Η εντολή για εκτύπωση ιστορικού είναι myHistory. Η επιλογή εντολής γίνεται μετά την εκτύπωση.
4) Οι ειδικοί χαρακτήρες  εκτός των wildcards (<,>,>>,&,|) πρέπει να διαχωρίζονται με space από την προηγούμενη και επόμενη λέξη (πχ. command < input > output)
5) Ο χαρακτήρας ";" δεν χρειάζεται να χωρίζεται από την προηγόυμενη λέξη.

Σημειώσεις - συνοπτική περιγραφή:

1) main.c:
    Η main αρχικά δημιουργεί τις δομές που θα κρατοούν τα aliases και το ιστορικό. Έπειτα ξεκινάει ένα infinite loop για να διαβάζει μια γραμμή από το stdin με την getline και να την προωθεί στην tokenise.
    Πρώτα το input χωρίζεται σε tokens ανά ';', ώστε να μπορούν αν τρέξουν σειριακά εντολές της μορφής:  ">> command1; command2;" . Το πρόγραμμα καλεί σε πολλά σημεία την strtok για να χωρίσει ένα σύνολο strings οπότε
    χρησιμοποιείται αντί αυτού η strtok_r() ώστε να μπορεί να της δίνεται σε κάθε κλήση της το ζητούμενο σημειό μνήμης και να αποφευχθεί για παράδειγμα η κλήση της strtok() στην tokenise() να τρέξει στο Input της main.

    Η promt() χρησιμοποιεί escape characters για να άλλαξει το χρώμα του promt.

    Αν η εντολή έτρεξε επιτυχώς η tokenise θα επιστρέψει 0 και θα τοποθετηθεί στο ιστορικό. Για να γίνει αυτό πρέπει να αντιγράψουμε το input πριν χωριστεί απο την strtok_r(), καθώς θα αλλάξει το αρχικό string και δεν θα μπορεί να περαστεί όπως ήταν στο ιστορικό.

2) tokenise.c:
    tokenise():

            Η tokenise αναλαμβάνει να χωρίσει την κάθε εντολή σε λέξεις και τις διαχειριστεί. Αρχικά αντιγράφεται η εντολή δύο φορές για τον ίδιο λόγο που αναφέρθηκε παραπάνω.
            Το πρώτο αντίγραφο χρησιμοποιείται στο δεύτερο loop χωρισμού της εντολής και το δεύτερο χρησιμοποιείται για να περαστεί ακέραιο το input στην pipe().
            Έπειτα χωρίζεται η εντολή ανα space για να καθοριστεί ο αριθμός λέξεων (argc) ώστε να γίνει allocate μνήμη για το διάνυσμα arguments (argv).
            Επαναλαμβάνουμε τον χωρισμό του input για να τοποθετήσουμε τις λέξεις στο argv. Για κάθε λέξη ελέγχουμε αν αποτελεί ειδικό χαρακτήρα (redirect,background,pipe ή wildcard) και ενημερώνουμε το αντίστοιχο flag και index.
            Στο τέλος πρέπει το argv να τελειώνει σε NULL για να περαστεί σωστά στην execvp().

            Έπειτα ελέγχεται αν η πρώτη λέξη είναι "createalias" ή "destroyalias" και καλούνται οι κατάλληλες συναρτήσεις. Αν η πρώτη λέξη είναι "myHistory" η κλήση της print_history() θα επιστρέψει την εντολή που επιλέχθηκε ή NULL αν δεν επιλέχθηκε καμία.
            Στην πρώτη περίπτωση καλείται αναδρομικά η tokenise() με όρισμα την εντολή που επιλέχθηκε.

            Τέλος ελέγχεται αν η πρώτη λέξη είναι κάποιο από τα αποθηκευμένα aliases.

            Αν βρέθηκαν wildcards χρησιμοποιείται η glob για να αποκτήσουμε πρόσβαση σε έναν πίνακα των ονομάτων των αρχείων που ανήκουν στο ζητούμενο σύνολο, και τον αριθμό αυτών τον αρχείων.
            Αν δεν βρέθηκαν, ο αριθμός αυτός τίθεται σε 1, ώστε να γίνει μια επανάληψη στο Loop που ακολουθεί.

            Για κάθε αρχείο: αν υπάρχουν redirect ή pipe χαρακτήρες η εκτέλεση περνά στις redirect() ή pipe_exec() αντίστοιχα. Σε άλλη περίπτωση καλείται η exec().

    exec():
            Η exec() καλεί fork() διότι η execvp() θα μεταφέρει την εκτέλεση εκτός του κώδικα και θα επιστρέψει μόνο σε περίπτωση σφάλματος. Η διεργασία παιδί που παράγεται καλεί την execvp() ενώ μητρική διεργασία περιμένει το παιδί αν δεν έχει δωθεί background character ή επιστρέφει αμέσως αν έχει.

3) redirect.c:

    which_red():
                Ελέγχεται αν μια λέξη αποτελεί redirection character και επιστρέφεται η κατάλληλη τιμή. Καθώς υπάρχει η περίπτωση να υπαρχούν input και output characters ταυτόχρονα το flag στην tokenise δουλεύει ως εξής:
                Αν είναι ίσο με 1 έχουμε μόνο output, αν είναι ίσο με 2 μόνο input και αν είναι ίσο με 3 έχουμε και τα δύο. Έτσι αθροίζοντας τις τιμές που επιστρέφει η which_red() καταλλήγουμε σε σωστή τιμή του flag.
                Για την συνάρτηση αυτή ο χαρακτήρας "<<" είναι το ίδιο με τον "<".

    redirect():
                Η redirect() καλεί fork() γιατί θα κληθεί η execvp(). Αναλόγως το flag ανοίγονται τα κατάλληλα αρχεία (input,output,input και output) και αλλάζει το αντίστοιχο stream με την dup2().
                Στην περίπτωση του output αν έχουμε "<<" το αρχείο που ανοίγει γίνεται με flag O_APPEND ώστε να γίνει η προσάρτηση σε υπάρχον αρχείο.
                Χρησιμοποιώντας τα indeces θέτουμε τον χαρακτήρα redirection και την επόμενη λέξη (τπ όνομα αρχείου) σε null ώστε να αγνοηθούν στην execvp().
                Το παιδί καλεί την execvp() με το νέο argv και ο γονέας περιμένει το παιδί αν δεν έχουμε background character και μετά κάνει reset τα input output streams.

4) pipe.c:

    pipe_exec():
                Αρχικά η συνάρτηση βρίσκει το index του pipe character και χωρίζει την εντολή σε δύο μέρη. Έπειτα δημιουργείται ένα pipe και καλούνται δυο fork(), ένα παιδί για κάθε μέρος της εντολής.
                Το πρώτο παιδί αλλάζει το output stream στο pipe και καλεί την tokenise για πρώτο μέρος , και το δεύτερο αλλάζει το input stream στο pipe και καλεί την tokenise για το δεύτερο μέρος.
                Η tokenise θα καλέσει κάποια στιγμή fork() οπότε πρέπει να κάνει exit() η διεργασία που την κάλεσε όταν τελειώσει. Καλείται η tokenise() αντί για execvp() ώστε να διαχειριστούν ενδεχόμενοι ειδικοί χαρακτήρες στα δύο μέρη.
                Η μητρική διεργασία περιμένει να τελειώσουν και επιστρέφει.

7) history.c/history.h:

        Η δομή myHistory περιλαμβάνει έναν πίνακα για  να αποθηκευτούν οι MAX_HISTORY πρόσφατες εντολές και έναν μετρητή αποθηκευμένων εντολών.

    add_history():
                    Προσθέτει μια επιτυχώς εκτελεσμένη εντολή στο ιστορικό.
                    Αν το ιστορικό είναι γεμάτο κάθε θέση στην λίστα αποθηκευμένων εντολών μετακινήται στην προηγούμενη θέση αφήνοντας ελεύθερη την τελευταία θέση και ελευθερώνοντας την λιγότερο πρόσφατη.
                    Αν δεν είναι γεμάτο η πιο πρόσφατη εντολή αποθηκεύεται στην τελευταία θέση του πίνακα και αυξάνεται ο μετρητής.

    print_history():

                    Εκτυπώνει τις αποθηκευμένες εντολές με αντίστροφη σειρά, και δίνει την δυνατότητα στον χρήστη να επιλέξει κάποια από αυτές πληκτρολογώντας τον αντίστοιχο αριθμό ή πατώντας enter να επιστρέψει στο shell.
                    Διαβάζει την απάντηση του χρήστη με την getline() και αν πατήθηκε enter επιστρέφει NULL ώστε να μην προχωρήσει σε εκτέλεση η tokenise.
                    Αν δώθηκε μη δεκτός αριθμός τυπώνεται αντίστοιχο μήνυμα αλλά δεν επιστρέφει η συνάρτηση ώστε να μπορεί ο χρήστης να επιλέξει έναν δεκτό αριθμό χωρίς να ξανακαλέσει την myHistory.
                    Αν δωθεί δεκτός αριθμός εκτυπώνεται " ">>" + εντολή " και επιστρέφεται στην tokenise η εντολή για να εκτελεστεί.

    free_his():
                Ελευθερώνει τις αποθηκευμένες εντολές.


8) alias.c/alias.h:

        Η δομή alias αποθηκεύει το όνομα του alias και την εντολή που αντιπροσωπεύει.
        Η δομή aliases αποθηκεύει όλα τα alias που δημιουργεί ο χρήστης και έναν μετρητή aliases.

    create_alias():
                    Αρχικά δεσμεύεται η μνήμη για ένα alias. Αν η λίστα είναι γεμάτη εκτυπώνεται αντίστοιχο μήνυμα. Αν δεν είναι δεσμεύεται η μνήμη για τα πεδία του καινούργιου alias, αντιγράφεται η εντολή που έδωσε ο χρήστης και αυξάνεται ο μετρητής.
                    Δεν ελέγχεται αν το όνομα αντιστοιχεί σε υπάρχουσα εντολή του συστήματος.
    check_alias():
                    Ελέγχει αν η πρώτη λέξη ένος input είναι το όνομα κάποιου αποθηκευμένου alias. Αν βρεθεί επιστρέφεται η έντολη, αν όχι επιστρέφεται NULL ώστε η tokenise να συνεχίσει την εκτέλεση.

    destroy_alias():
                    Πρώτα βρίσκεται η θέση του δεδομένου alias στη λίστα. Αν είναι το τελευταίο απλώς ελευθερώνεται ενώ αν είναι σε οποιαδήποτε άλλη θέση αντικαθήσταται από το τελευταίο.

    free_alias():
                    Ελευθερώνει τα πεδία των αποθηκευμένων aliases.
